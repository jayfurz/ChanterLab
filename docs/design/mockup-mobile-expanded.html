Mobile · expanded — Practice / Sound / More

One drawer (the existing transport), three tabs. Tabs are clickable in this mockup. Spec §3.

1
The score never disappears. Panes are capped at min(46vh, 420px) with overflow-y:auto — vs. today's 12–14 rows at 70vh overflow:hidden that cover the whole score and clip the Sections row unreachably (measured 67px clip on the Complete Liturgy with mic + save chip).
2
Tab strip reuses the app's .seg/.segbtn pattern — no new visual vocabulary. Default tab is Practice on every load. Contextual rows (Verse, Sections, Rec mix) hide exactly as today.
3
Grouping by frequency: Practice = touched while practicing (voice, verse, tempo, loop, sections). Sound = what you hear/capture (mic, headphones, my-part, instrument, record). More = persisted taste + piece meta (library, scoring strictness, view). Row markup and every element id are unchanged — rows are moved, not rebuilt.
